# Development Summary: 2026-06-03

Implemented **all five phases** of `master_roadmap.md` end-to-end. Heavy platform
deps (frida, pymem, scapy, ray, instructor, grpcio, mlagents, godot-rl) are kept
optional behind import guards with deterministic offline fallbacks, so the SDK
runs today and lights up the integrations as packages are installed. Per-task
notes live in `development/notes/`.

- New `SchemaClassifier` infers type/min/max/role/importance + `normalize` block
  from sampled telemetry frames; unifies the `state_map.json` contract across
  strategies (Task 1.4).

  loopback proxy, nested-JSON flattening with recorded json paths (Task 1.1).
- `FridaHookingStrategy`: live `on("message")` frame collection + field offsets
  (Task 1.2). `MemoryScrapingStrategy`: pattern scan → base ptr → sampled floats
  with offsets (Task 1.3).

## Phase 5 — Engine Bridges (`src/engine_bridges.py`)

# MVP: Live Roguelike QA Swarm (demo target: Caves of Qud)

Pivoted from the synthetic-only proof-of-concept to a **real game connection**.
Plan: `~/.claude/plans/linear-soaring-pixel.md`; PRD: `PRD.md`. Per-step notes:
`development/notes/mvp_*.md`.

### Engine reality (drove the choices)
- Inspected both installed games: **Caves of Qud = Unity/Mono** (`MonoBleedingEdge`,
  `Assembly-CSharp.dll` 11.5 MB), **Balatro = LÖVE/Lua** (`love.dll`/`lua51.dll`).
- **Neither is a native binary**, so Ghidra-MCP is the *native-roguelike* rung, not
  for these two. Demo target chosen: **Caves of Qud**.

### Built (verified headless via a protocol-accurate mock)
- **Connector ladder** `src/connectors/`: `base` (Connector/GameTransport/StateFrame),
  `socket_transport` (length-prefixed JSON), `dotnet_connector` (active CoQ rung;
  curated schema + optional pythonnet reflection), `ghidra`/`lua`/`vision` scaffolds.
- **`src/orchestrator.py`** — engine detection → connector → `state_map.json` + transport.
- **`src/live_game_env.py`** — Gym env over a transport, reusing reward_generator + ICM + logger.
- **`src/swarm.py`** + `configs/coq_qa.yaml` — N PPO testers with distinct reward profiles.
- **`tools/mock_coq_server.py`** — 80×25 zone sim with injected softlock/lava/crash faults.
- **`src/dashboard.py`** — multi-agent roster (`/api/swarm` + card).
- Action count now flows to the ICM (`reward_generator` `action_dim`); `SDKConfig`
  accepts an in-memory `raw_config` for per-agent overrides.

### Shipped as source (user-enabled)
- **`mods/CoQ_QA_Bridge/`** — complete Harmony mod (TCP listener + `XRLCore.PlayerTurn`
  postfix pump) speaking the same wire protocol as the mock. `--transport live`
  needs zero Python changes.

### Verification
- Orchestrator on the real CoQ install → `unity-mono`, 9-dim `state_map.json`.
- 3-tester swarm × 8k steps over the mock surfaced **all 3 fault classes**
  (`HAZARD_DAMAGE`, `SOFTLOCK_SUSPECTED`, `CRASH_NUMERIC`) across 3 consecutive
  clean runs; wrote per-tester heatmaps/reports + `swarm_report.json`. No single
  tester finds everything — the swarm union does.
- Profiles: two PPO testers + one pure-random fuzzer (`chaos_monkey`). Reliability
  came from rewarding each distinct fault once (no farming) + persistent novelty.
- Dashboard `/api/swarm` roster renders; synthetic `src/train` unchanged (no regression).
- Test hygiene: the mock uses SO_REUSEADDR; run a single instance and clear the
  port between runs (stale racing servers otherwise corrupt results).

### Tech debt / next
- CoQ mod can't be compiled headless here — needs the game + enable step.
- Ghidra/Lua/Vision rungs are interface-correct scaffolds (runtime paths raise
  actionable `NotImplementedError`): next milestones are Ghidra→pymem pointer maps,
  Lovely Lua bridge (Balatro), and dxcam+VLM vision.
